# ハンズオン講師ガイドライン

## 基本姿勢

### 1. 明確な説明と深い理解の促進
- 基本的な実装方法は明確に説明する
  ```example
  "ActivitySourceは分散トレーシングの基本コンポーネントで、以下の理由で使用します：
  1. 処理の追跡：アプリケーション全体の処理フローを可視化
  2. パフォーマンス測定：各処理にかかる時間を正確に計測
  3. エラー追跡：問題発生時の状況を詳細に記録
  
  詳細は公式ドキュメント（URL）を参照してください"
  ```
- エラー発生時は原因と解決方法を丁寧に説明
  ```example
  "このエラーは自動計装の設定が不足しているために発生しています。
  ASP.NET Coreの場合、以下の理由でこの設定が必要です：
  1. HTTPリクエストの自動追跡
  2. ミドルウェアの処理時間の計測
  3. エラー情報の自動収集
  
  実装方法の詳細は公式ガイド（URL）で確認できます"
  ```
- 実装の意図や理由を確認し、理解を深める
  ```example
  ❌: "なぜその実装にしたのか聞くだけ"
  ⭕: "その実装方法を選んだ理由を伺い、より良い方法がある場合は、
      なぜそちらの方が適しているのかを具体的な理由と共に説明する"
  ```

### 2. セッション運営と実装サポート

#### 基本的なセッション構成（60分）
a. 導入と説明（15分）
   ```example
   "今回は〇〇を実装します。
    まず基本的な実装方法を説明し、その後で実際に実装を進めていきましょう。
    特に△△の部分が重要になります"
   ```
   - 学習目標の共有
   - 実装方法の説明
   - 参考資料の共有

b. 実装時間（30分）
   ```example
   "それでは実際に実装を始めましょう。
    画面を共有しながら進めていただき、気になる点があればその場で確認していきます"
   ```
   - 画面共有での実装
   - ペアプログラミング形式のサポート
   - 実装中の質疑応答

c. 確認と振り返り（15分）
   ```example
   "実装いただいた内容を確認していきましょう。
    特に〇〇の部分の実装意図について教えていただけますか？"
   ```
   - 実装結果の確認
   - コードレビュー
   - 学習内容の共有
   - 次回への準備

- 実装中は画面共有で一緒にコードを確認
  ```example
  "実装中の画面を見せていただけますか？一緒にコードを確認しながら進めていきましょう"
  ```

- 実装結果を確認しながら理解を深める
  ```example
  "今の実装で、トレースがどのように記録されているか確認してみましょう。
   期待通りの情報が取得できていますか？"
  ```

- コードレビューを通じた学習
  ```example
  "実装したコードを見ながら、なぜこの方法を選んだのか、
   他の方法との比較などについて一緒に考えてみましょう"
  ```

- 休憩を取りながら段階的に進める
  ```example
  "基本設定の実装ができましたね。
   ここで一度休憩を取って、次はカスタム計装の実装に進みましょうか？"
  ```

- 予定時間を考慮した柔軟な進行
  ```example
  "この部分の実装と動作確認にはもう少し時間が必要そうですね。
   今日はここまでにして、次回じっくり確認しながら進めましょう"
  ```

### 3. フィードバックの収集と提供
- 実装中の気づきや疑問をその場でメモするよう促す
  ```example
  "気になった点や疑問に思ったことは、その場でメモしておくと振り返りの時に役立ちますよ"
  ```
- セッション終了時には必ず振り返りの時間を設ける
  ```example
  "今回の実装を通じて、特に印象に残った点はありますか？"
  ```
- 具体的な改善点だけでなく、良かった点も積極的に共有
  ```example
  "エラーメッセージを丁寧に確認して原因を特定できていましたね"
  ```
- 次回セッションでの目標を学習者と一緒に設定
  ```example
  "次回はどの部分に焦点を当てて進めていきたいですか？"
  ```

### 4. つまずきへの対応
- エラーや問題は学習の機会として捉える姿勢を示す
  ```example
  "このエラーは良い学びの機会ですね。一緒に原因を探ってみましょう"
  ```
- 焦らず、一つずつ問題を切り分けることを提案
  ```example
  "まずはエラーメッセージを確認して、どの部分で問題が起きているか特定してみましょうか"
  ```
- 必要に応じてデバッグツールの使い方もレクチャー
  ```example
  "この場合、デバッガーを使うと問題の原因が見つけやすいかもしれません"
  ```
- 解決後は同様の問題に対処できるよう、気づきを共有
  ```example
  "今回のエラーの解決方法は、似たような状況でも使えそうですね"
  ```

### 5. モチベーション管理
- 小さな進捗も積極的に評価
  ```example
  "テストケースが通りましたね！一つずつ着実に進んでいますよ"
  ```
- 難しい概念は身近な例えを用いて説明
  ```example
  "トレースは、料理のレシピのように各ステップの実行を記録するようなものです"
  ```
- 実務での活用シーンを具体的に示す
  ```example
  "このモニタリング設定は、本番環境でのトラブルシューティングで特に役立ちます"
  ```
- 学習者のペースを尊重し、焦らせない
  ```example
  "理解を深めることが大切なので、じっくり進めていきましょう"
  ```

### 6. 継続的な改善
- 各セッション後に講師自身の振る舞いを振り返る
  ```example
  "説明が分かりやすかったか、サポートのタイミングは適切だったかを確認"
  ```
- 学習者からのフィードバックを基に指導方法を調整
  ```example
  "前回のフィードバックを受けて、今回は実装時間を多めに確保しました"
  ```
- 教材や進め方の改善点を記録
  ```example
  "この部分の説明は図を追加すると分かりやすくなりそう"
  ```
- 定期的に他の講師と知見を共有
  ```example
  "効果的だった説明方法や、よくある躓きポイントについて情報交換"
  ```

## セッション毎のチェックリスト

### セッション開始前
- [ ] 学習目標の明確化
- [ ] 必要な教材の準備
- [ ] 前回の振り返り内容の確認
- [ ] 想定される躓きポイントの洗い出し

### セッション中
- [ ] 時間配分の管理
- [ ] 学習者の理解度の観察
- [ ] 適切なタイミングでの質問・アドバイス
- [ ] 休憩時間の確保

### セッション終了時
- - [ ] 振り返りの実施
  ```example
  "今回の実装を通じて学んだことをまとめてみましょう。
   特に重要だと感じた点はありましたか？"
  ```
- [ ] 学習内容のコミット
  ```example
  "今回の実装内容をコミットしておきましょう。
   コミットメッセージには実装した機能と学んだポイントを含めると良いですね"
  ```
- [ ] 変更箇所の確認
  ```example
  "コミット前に変更した箇所を一緒に確認しましょう。
   git diffで変更内容を見ながら、実装のポイントを振り返りましょうか"
  ```
- [ ] フィードバックの収集
- [ ] 次回の目標設定
- [ ] 宿題・予習項目の確認

## フィードバック収集のポイント

### 実装プロセス
- 躓いた箇所とその原因
- 理解が深まった点
- 追加で確認したい事項
- 実装時の工夫点

### 理解度
- 概念の理解度
- 実装スキルの向上
- トラブルシューティング能力
- 応用力の獲得

### 学習環境
- 教材の分かりやすさ
- セッションの進め方
- サポートの適切さ
- 時間配分の妥当性
